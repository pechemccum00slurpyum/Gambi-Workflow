Guidelines for managing bonus programs, monitoring for abuse, and assisting CS agents with bonus inquiries.

Summary

The Bonus Team SOP covers procedures for applying bonuses, detecting potential abuse, and verifying eligibility.

Key Advice

  • Verify promotion details before adding bonuses.
  • Tag KYC Team if multiple accounts are suspected.
  • Assist CS with user inquiries related to specific promotions.

Workflow Diagram

                flowchart TD
    A[Receive Inquiry from CS] --> B[Check User Account for IP Country, IP Relations, Same Documents, Usernames, High-Risk Emails]
    
    B --> C{Any Matches Detected?}
    
    %% Matches Detected Workflow
    C -->|Yes| D[Investigate Potential Abuse Among Related Users]
    D --> E{Is There Matching ID Across Related Users?}
    
    %% Permanent Ban for ID Match
    E -->|Yes| F[Ban All Related Users Permanently; Document Reason in User's Notes]
    
    %% Temporary Ban and Document Verification for Non-ID Matches
    E -->|No| G[Temporarily Ban All Users; Request Selfie with ID and Gambi Paper]
    G --> H[User Submits Documentation]
    H --> I[Review Submitted Documents]
    
    %% Final Decision Based on Document Check
    I --> J{Does Submitted Document Match Any Other Related Users' ID?}
    J -->|Yes| F
    J -->|No| K[Unban Users and Allow Continued Account Usage; Document Decision in User's Notes]
    
    %% No Matches Detected Workflow
    C -->|No| L[No Abuse Detected; Allow User to Continue]
    
    %% Escalation for Uncertain Cases
    B -->|Uncertain Case| M[Escalate to Higher Department for Further Review]

    %% Outcome and Closure
    F --> N[Document Decision and Close Inquiry]
    K --> N
    L --> N
    M --> N
